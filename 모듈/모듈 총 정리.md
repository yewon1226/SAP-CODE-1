## SD 모듈
</br>

- 기업이 고객에게 제품이나 서비스를 판매하고, 이를 배송하며, 최종적으로 대금을 청구하는 모든 비즈니스 프로세스를 관리하는 모듈
</br>

### SD 주요 프로세스: Order-to-Cash (O2C)
```
Inquiry(제품 문의) → Quotation(견적 제공) → Sales Order(주문 생성) → Delivery(출고 문서) → PGI(실제 재고 출고) → Billing(청구서) → Payment(결제 처리)
```

**1. 영업 전 단계 (Pre-Sales)** : 고객의 문의(Inquiry)에 응대하고 견적서(Quotation)를 발송

**2. 판매 주문 (Sales Order)** : 고객의 구매 의사가 확정되면 시스템에 주문을 생성. 이때 가격(Pricing), 재고 확인(ATP), 신용 한도(Credit Limit) 등이 자동으로 체크.

**3. 출하 처리 (Delivery/Shipping)** : 배송 문서 생성(Delivery Creation).

**4. Picking & Packing** : 창고에서 물건을 꺼내 포장.

**5. Post Goods Issue (PGI)** : 출하 확정. 이 시점에 재고가 감소하고 회계상 매출 원가가 발생.

**6. 대금 청구 (Billing)** : 고객에게 인보이스(청구서)를 발행하고 매출을 인식.

**7. 입금 처리 (Payment)** : 고객이 보낸 대금을 확인하여 채권을 반제(Clear).
