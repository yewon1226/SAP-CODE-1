## SD 모듈
- 기업이 고객에게 제품이나 서비스를 판매하고, 이를 배송하며, 최종적으로 대금을 청구하는 모든 비즈니스 프로세스를 관리하는 모듈
</br>

### < SD 주요 프로세스: Order-to-Cash (O2C) >
```
Inquiry(제품 문의) → Quotation(견적 제공) → Sales Order(주문 생성) → Delivery(출고 문서) → PGI(실제 재고 출고) → Billing(청구서) → Payment(결제 처리)
```

**1. 영업 전 단계 (Pre-Sales)** : 고객의 문의(Inquiry)에 응대하고 견적서(Quotation)를 발송

**2. 판매 주문 (Sales Order)** : 고객의 구매 의사가 확정되면 시스템에 주문을 생성. 이때 가격(Pricing), 재고 확인(ATP), 신용 한도(Credit Limit) 등이 자동으로 체크.

**3. 출하 처리 (Delivery/Shipping)** : 배송 문서 생성(Delivery Creation).

**4. Picking & Packing** : 창고에서 물건을 꺼내 포장.

**5. Post Goods Issue (PGI)** : 출하 확정. 이 시점에 재고가 감소하고 회계상 매출 원가가 발생.

**6. 대금 청구 (Billing)** : 고객에게 인보이스(청구서)를 발행하고 매출을 인식.

**7. 입금 처리 (Payment)** : 고객이 보낸 대금을 확인하여 채권을 반제(Clear).

</br>
</br>

### < 핵심 구성요소 (Key Components) >
**1. 마스터 데이터** → 고객, 제품, 가격 등 판매 과정에서 반복적으로 사용하는 기본 정보 데이터

  - 자재 마스터 (Material Master) → 판매할 제품의 기본 정보(제품코드, 판매단위, 배송조건 등)를 관리하는 데이터
  - 거래처 마스터 (Customer Master / Business Partner) → 고객의 주소, 결제조건, 배송조건 등 거래처 정보를 관리하는 데이터
  - 가격 마스터 (Condition Master) → 제품 가격, 할인, 세금 등 판매 가격을 결정하는 조건 정보를 관리하는 데이터

**2. 출하 및 운송 관리 (Shipping & Transportation)** → 어떤 운송 수단을 쓸지, 운송 경로는 어떻게 되는지(Route) 관리

**3. 신용 관리 (Credit Management)** → 고객이 외상으로 물건을 가져갈 때, 미수금이 너무 많으면 주문이 생성되지 않도록 막는 신용 통제 기능

</br>
</br>

### < 타 모듈과의 연계성 >
- `SD ↔ MM` (자재): 주문 시 재고가 있는지 확인(ATP Check)하고, 출하 시 재고를 감소시킴
- `SD ↔ PP` (생산): 재고가 없으면 생산 요청(MTO)을 보내 물건을 만들게 함
- `SD ↔ FI` (재무): 물건을 팔았을 때 매출 전표를 생성하고 외상값을 관리
- `SD ↔ CO` (관리): 판매 수익성을 분석하여 어떤 제품이 돈이 되는지 계산

</br>
</br>

## MM 모듈
- 기업의 비즈니스 운영에 필요한 원재료, 부자재, 완제품 등을 조달(구매)하고, 창고 내 재고를 관리하며, 공급업체로부터 받은 인보이스를 처리하는 모듈
</br>

### < MM 주요 프로세스: Procure-to-Pay (P2P) >
```
Purchase Requisition(구매 요청) → Purchase Order(구매 주문) → Goods Receipt(입고 처리) → Invoice Verification(송장 검증) → Payment(지불 처리)
```

**1. 구매 요청 (Purchase Requisition)** : 현업 부서(생산, 영업 등)에서 필요한 자재를 사달라고 구매 부서에 요청.

**2. 구매 주문 (Purchase Order)** : 구매 요청을 검토하여 공급업체(Vendor)에 정식으로 발주를 넣음. 이때 가격, 납기일, 수량 등이 확정.

**3. 입고 처리 (Goods Receipt)** : 공급업체로부터 물건이 도착하면 시스템에 입고를 잡음. 이 시점에 창고의 재고 수량이 증가.

**4. 송장 검증 (Invoice Verification)** : 공급업체가 보낸 인보이스(송장)의 가격과 수량이 구매 주문 및 입고 내역과 일치하는지 대조.

**5. 지불 처리 (Payment)** : 검증이 완료된 인보이스에 대해 재무 부서에서 대금을 지급. (FI 모듈 연동)

</br>
</br>

### < 핵심 구성요소 (Key Components) >

**1. 마스터 데이터** → 구매 및 재고 관리 과정에서 핵심이 되는 기본 정보 데이터

  - 자재 마스터 (Material Master) → 자재의 기본 규격, 창고 위치, 평가 가격(단가) 등을 관리하는 데이터
  - 공급업체 마스터 (Vendor Master / BP) → 물건을 사오는 거래처의 주소, 지불 조건, 통화 등을 관리하는 데이터
  - 구매 정보 레코드 (Info Record) → 특정 자재와 특정 공급업체 간의 거래 조건(최근 가격, 리드타임 등)을 저장한 데이터

**2. 재고 관리 (Inventory Management)** → 창고 내 재고의 이동(입고, 출고, 이관)을 관리하고 재고 실사를 수행

**3. 자재 소요량 계획 (MRP)** → 현재 재고와 생산 계획을 분석하여, 필요한 자재를 언제 얼마나 사야 할지 자동으로 계산하는 기능

</br>
</br>

### < 타 모듈과의 연계성 >
- `MM ↔ SD` (영업): 영업에서 판매할 재고가 있는지 확인하고, 출하 시 MM의 재고 데이터를 차감함
- `MM ↔ PP` (생산): 생산에 필요한 원재료를 적기에 공급하고, 생산이 완료된 제품을 창고에 입고함
- `MM ↔ FI` (재무): 입고 시 자재 자산 가치를 반영하고, 공급업체 대금 지급을 위한 송장 데이터를 전달함
- `MM ↔ CO` (관리): 자재의 원가(Standard Price 등) 정보를 제공하여 제품의 제조 원가 계산을 도움
- `MM ↔ QM` (품질): 자재 입고 시 자동으로 품질 검사 로트(Inspection Lot)를 생성하며, 검사 합격 판정이 나기 전까지 해당 자재를 사용할 수 없도록 제어함

</br>
</br>

## PP 모듈
- 기업의 제조 활동을 계획하고 실행하는 모듈로, 판매 계획에 맞춰 제품을 생산하기 위한 자재 소요량을 계산하고 생산 공정을 관리하는 모듈

</br>

### < PP 주요 프로세스: Plan-to-Produce >
```
Demand Management(수요 관리) → MPS(기준 생산 계획) → MRP(자재소요량 계획) → Production Order(생산 오더 생성) → Goods Issue(원재료 출고) → Confirmation(생산 실적 확정) → Goods Receipt(완제품 입고)
```

**1. 수요 관리 (Demand Management)** : 영업(SD)의 판매 계획이나 과거 데이터를 바탕으로 제품이 얼마나 필요할지 예측하여 생산 계획을 수립.

  - MTS (Make-to-Stock): 판매 예측을 바탕으로 재고를 미리 쌓아두기 위해 생산하는 방식 (기성품)
  - MTO (Make-to-Order): 고객의 주문이 들어온 후에 생산을 시작하는 방식 (주문 제작)

**2. MPS (Master Production Schedule, 기준 생산 계획)** : 판매 계획을 바탕으로 최종 완제품의 생산 일정과 수량을 확정하는 단계. 

**3. MRP (Material Requirements Planning, 자재소요량 계획)** : MPS에서 확정된 완제품 수량을 만들기 위해 필요한 하위 부품 및 원재료가 무엇인지, 얼마나 부족한지 계산하여 구매/생산을 제안하는 과정.

**4. 생산 오더 생성 (Production Order)** : "언제, 어떤 라인에서, 몇 개를 만들라"는 구체적인 작업 지시서를 시스템에 생성.

**5. 자재 출고 (Goods Issue)** : 제품을 만들기 위해 창고에서 원재료나 부품을 생산 라인으로 가져감. (MM 재고 감소)

**6. 실적 확정 (Confirmation)** : 생산이 완료되면 실제로 몇 개를 만들었는지, 시간은 얼마나 걸렸는지 시스템에 입력.

**7. 완제품 입고 (Goods Receipt)** : 만들어진 제품을 창고에 넣음. 이 시점에 완제품 재고가 증가. (MM 재고 증가)

</br>
</br>

### < 핵심 구성요소 (Key Components) >
**1. 마스터 데이터** → 생산 체계를 구축하는 가장 기초적인 데이터

  - BOM (Bill of Materials) → 제품 하나를 만드는 데 필요한 부품 리스트와 수량 정보 (레시피와 유사)
  - 작업장 (Work Center) → 생산이 수행되는 기계, 라인, 사람 등의 물리적 장소
  - 공정 (Routing) → 제품이 완성되기까지 거쳐야 하는 순서와 각 단계별 표준 작업 시간

**2. 생산 계획 (Production Planning)** → 장기적인 판매 목표와 현재 공장의 생산 능력을 고려하여 생산 일정을 최적화

**3. 용량 계획 (Capacity Planning)** : 기계나 인력이 감당할 수 있는 작업량인지 확인하여 과부하를 방지

</br>
</br>

### < 타 모듈과의 연계성 >
- `PP ↔ SD` (영업): 고객의 주문 정보를 받아 생산 계획을 세우고, 완성된 제품 정보를 영업에 전달함
- `PP ↔ MM` (자재): 생산에 필요한 원재료 재고를 확인하고, 생산 후 완제품을 창고로 입고함
- `PP ↔ FI/CO` (재무/관리): 생산 과정에서 발생한 노무비, 기계 가동비 등을 계산하여 제품의 정확한 제조 원가를 산출함
- `PP ↔ QM` (품질): 생산된 제품이 기준에 맞는지 검사하고 합격된 제품만 입고하도록 제어함

</br>
