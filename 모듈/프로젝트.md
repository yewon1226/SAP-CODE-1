### MM
```
PR → PO → GR → Invoice
구매요청 → 구매오더 → 입고 → 송장
```
1. 자재 마스터 데이터를 조회하고 자재 기본정보와 플랜트별 자재 정보를 관리·확인하는 화면
2. 공급업체(거래처) 정보를 조회하고 해당 공급업체의 구매정보(구매 이력·조건 등)를 확인하는 화면
< PR >
3. 구매요청(PR) 목록을 조회하고, 해당 요청에 대해 승인 요청을 하거나 상태를 확인하는 화면
4. 구매요청(PR)을 조회하고 승인 처리하면서 동시에 공급업체(거래처)를 선택해 구매 진행을 준비하는 화면
< PO >
5. 승인된 구매요청(PR)을 기반으로 구매오더(PO)를 생성하고 조회하는 화면
< GR >
6. 구매오더(PO)를 기준으로 자재를 창고에 입고 처리(GR)하고 입고된 자재문서를 생성·조회하는 화면
7. 입고·출고 등 자재 이동 기록이 담긴 자재문서를 조회하고 상세 내역을 확인하는 화면
< QM 품질관리 >
8. 입고된 자재가 품질 기준에 맞는지 검사하는 단계 ( QM )
9. 불량 자재를 공급업체로 다시 반품하는 처리
< Invoice >
10. 구매오더 기준으로 송장(Invoice)을 확인하고 금액·수량을 검증하는 송장검증 화면
11. 송장(Invoice) 관련 문서를 조건에 따라 조회하고 송장 정보를 확인하는 화면
< 기타 >
- 플랜트·창고 간 자재를 이동하여 재고 위치를 변경하는 재고 이동(Stock Transfer) 처리 화면
- 재고 부족이나 생산 계획을 기준으로 시스템이 자동으로 구매요청(PR)을 생성하는 기능 (플랜트·창고별 재고, 안전재고, 부족 자재를 실시간으로 확인하는 Fiori 대시보드)
- 생성된 PO의 수량, 가격, 납기일 등을 수정하는 관리 화면
- 입고된 자재가 품질 기준에 맞는지 검사하는 단계 ( QM )
- 불량 자재를 공급업체로 다시 반품하는 처리
- 플랜트·창고별 현재 재고 수량을 조회하는 화면
- 모바일이나 웹에서 구매요청을 바로 승인·반려할 수 있는 Fiori 앱 (긴급시)
- 공급업체 성과 분석 (납기, 가격, 품질 기준으로 공급업체 성과를 분석하는 Fiori 분석 화면)
