# 실전을 위한 SQL 무료 강의
### 실전을 위한 SQL 무료 강의 (1/18) : Select From
</br>

- 특정 테이블에서 원하는 열을 선택해 출력
```sql
SELECT 출력하고 싶은 결과
FROM   테이블 이름
```
</br>

### 실전을 위한 SQL 무료 강의 (2/18) : Select * From
</br>

- 해당 테이블의 모든 열을 출력
```sql
SELECT *
FROM   테이블 이름
```

### 실전을 위한 SQL 무료 강의 (3/18) : Where (1)
</br>

- 조건에 맞는 행만 조회
```sql
SELECT 출력하고 싶은 결과
FROM   테이블 이름
WHERE  조건문
```
</br>

---

### 실전을 위한 SQL 무료 강의 (4/18) : Where (2)
</br>

- 두 개 이상의 조건을 써야할 때는 AND로 연결
```sql
WHERE 나라 = '멕시코' AND 나이 = 30
```
</br>

- 여러 조건 중에 한 조건만 만족해도 될 때는 OR로 연결
```sql
WHERE 나라 = '멕시코' OR 나이 = 30
```
</br>

- 제외하고 싶은 필터가 있을 때는 != 나 <>을 사용
```sql
WHERE 나라 != '멕시코'
WHERE 나라 <> '멕시코'
```
</br>

- 크다는 >, 작다는 <, 크거나 같다는 >=, 작거나 같다는 <=
```sql
WHERE 나이 >= 30 AND 나이 < 40
```
</br>

- 문자는 '', 정수는 그냥!
```sql
WHERE 나라 = '멕시코' OR 나이 = 30
```
</br>

- 같은 열을 이용한 여러 조건은 in / NOT in 과 리스트를 이용
```sql
WHERE 나라 in ('멕시코', '미국', '한국', '영국', '일본')
WHERE 나라 NOT in ('멕시코', '미국', '한국', '영국', '일본')
```
</br>

### 실전을 위한 SQL 무료 강의 (5/18) : Between, Like
</br>

### Where 사용법 (between)
- 범위를 나타낼 때는 BETWEEN을 사용하면 유용
```sql
WHERE 나이 BETWEEN 30 AND 39    // WHERE 나이 >= 30 AND 나이 <= 39 (나이가 30대인 사람만 출력)
```
</br>

- 제외하고 싶은 범위는 NOT BETWEEN을 사용
```sql
WHERE 나이 NOT BETWEEN 30 AND 39    // WHERE 나이 < 30  OR 나이 >= 40 (나이가 30대인 사람만 제외하고 출력)
```
</br>

### Where 사용법 (Like with %)
- 일정 패턴을 찾을 때는 Like/NOT LIKE 연산자가 유용
```sql
WHERE 이름 LIKE 'Tom%'        // 이름이 Tom으로 시작하는 사람을 출력하고 싶다면
WHERE 이름 NOT LIKE '%y'      // 이름이 y로 끝나지 않는 사람을 출력하고 싶다면
WHERE 이름 LIKE '%or%'        // 이름에 or이 어디든 들어있는 사람을 찾으려면 (처음이나 끝이어도 상관없음)
```
</br>

### 실전을 위한 SQL 무료 강의 (6/18) : Order By
</br>

- 조건에 맞는 결과를 특정 열 기준으로 정렬
- ASC: 오름차순, DESC: 내림차순 (생략시 ASC)
```sql
SELECT   출력하고 싶은 결과
FROM     테이블 이름
WHERE    조건문
ORDER BY 열 이름 ASC (DESC)
```
</br>

- 여러 개의 열 이름을 이용해 정렬하고 싶을 때는 콤마로 구분
```sql
ORDER BY 고객가입일 DESC, 고객이름 ASC
```
</br>

### 실전을 위한 SQL 무료 강의 (7/18) : Count
</br>

- 조건에 맞는 데이터 개수(행 수) 출력
- 집계함수 중 하나
- COUNT(*) 혹은 COUNT(열 이름)으로 사용
- 뒤에 AS를 붙이고 이 열의 새로운 이름을 적음
```sql
SELECT  COUNT(*)
FROM    테이블 이름
WHERE   조건문
```
</br>

- COUNT(DISTINCT 열이름)을 사용하면 unique한 행의 수를 셈
- SELECT 결과에서 중복된 값을 제외하고 고유한 값만 반환
```sql
SELECT COUNT(customer_id) as total_customers,
       COUNT(DISTINCT customer_id) as total_unique_customers
FROM Customers
```
</br>

### 실전을 위한 SQL 무료 강의 (8/18) : SUM, AVG, MIN, MAX
</br>

- sql에서 쓰이는 집계함수 중 하나
- SUM : 총 합계를 계산 (빈 행 제외)

- MIN : 최소값을 계산
- MAX : 최대값을 계산
```sql
SELECT  SUM(열이름)
FROM    테이블 이름
WHERE   조건문
```

### 실전을 위한 SQL 무료 강의 (9/18) : Group By
</br>

- 열 이름 : 그룹핑 하고 싶은 열
- SUM(열이름) : 계산하고 싶은 열 (숫자)
- WHERE 다음에 씀. (ORDER BY 전에 씀)
```sql
SELECT   열 이름, SUM(열이름)
FROM     테이블 이름
WHERE    조건문
GROUP BY 열 이름
```
</br>

- 대부분의 경우에는 SELECT 뒤에 나오는 열과 GROUP BY 뒤에 나오는 열이 같지만, 아닐수도 있다. SELECT 뒤의 열 이름은 반드시 GROUP BY 뒤에 쓴 열 이름 중 하나를 써야 함.
- SELECT 뒤에 나오는 열이름(GROUP BY 뒤의 열 이름 중이어야 함) 외에는 집계함수를 사용
- GROUP BY는 그룹별 갯수, 합계, 평균, 최소값, 최대값 등등을 구하기 위함임
```sql
SELECT   Country, City, SUM(Sales)         // 가능
FROM     Customers
GROUP BY Country, City                     

SELECT   Country, SUM(Sales)               // 가능
FROM     Customers
GROUP BY Country, City                           

SELECT   Country, City, SUM(Sales)         // 불가능
FROM     Customers
GROUP BY Country                          
```
</br>

### 실전을 위한 SQL 무료 강의 (10/18) : Having
</br>

- GROUP BY로 묶은 집계 결과에 대한 조건
- GROUP BY 다음에 씀 (ORDER BY 전에 씀)
- WHERE : GROUP BY 전에 필터링 하는 것, HAVING : GROUP BY 결과를 필터링 하는 것
```sql
SELECT   열 이름, SUM(열이름)
FROM     테이블 이름
WHERE    조건문
GROUP BY 열 이름
HAVING   조건문 (집계결과에 대해)
```
</br>

### 실전을 위한 SQL 무료 강의 (11/18) : As
</br> 

- 열이나 테이블에 별칭 붙이기 (테이블은 JOIN 사용할 때 흔하게 쓰임)

```sql
SELECT  열 이름 AS 새로운 이름

```
</br>

- As는 생략 가능
```sql
SELECT  고객이름 이름, 고객주소 주소
FROM    고객표
```

### 실전을 위한 SQL 무료 강의 (12/18) : CASE WHEN
</br>

- 기존의 열과 조건문을 이용해서 새로운 열을 만들고 싶을 때 사용
- 조건문은 순차적으로 진행. 첫번째 조건을 만족하면 다음 조건들을 확인하지 않고 바로 첫번째 이름 부여.
- 마지막 ELSE는 생략 가능 (END는 생략하지 않음)
```sql
CASE
       WHEN 조건1 THEN '이름1'
       WHEN 조건2 THEN '이름2'
       WHEN 조건3 THEN '이름3'
       ELSE '이름4' END
       AS 새로운열이름
```
```sql
SELECT 
       이름,
       점수,
       CASE
            WHEN 점수 >= 90 THEN 'A등급'
            WHEN 점수 >= 80 THEN 'B등급'
            WHEN 점수 >= 70 THEN 'C등급'
            WHEN 점수 >= 60 THEN 'D등급'
            ELSE 'F등급'
       END AS 성적등급
FROM 학생테이블;
```
</br>

### 실전을 위한 SQL 무료 강의 (13/18) : Inner Join
</br>

- INNER JOIN은 테이블1과 테이블2 공통으로 있는 행만 출력
- INNER를 생략하고 JOIN 이라고만 써도 됨
```sql
SELECT      테이블1.열이름, 테이블2.열이름
FROM        테이블 1
INNER JOIN  테이블 2
ON          테이블1.열이름 = 테이블2.열이름
```
</br>

- 테이블이 여러 개 나오므로 AS를 이용
```sql
SELECT      표1.고객이름, 표2.세일
FROM        고객표 AS 표1
INNER JOIN  주문내역 AS 표2
ON          표1.고객번호 = 표2.고객번호
```
</br>

- SELECT 뒤에 열 이름을 사용할 때 같은 열 이름이 두 개 이상의 테이블에 있을 때는 테이블의 출처를 표시해야 함 (같은 열 이름이 없을 경우에는 생략 가능)
```sql
SELECT      고객이름, 세일
FROM        고객표 AS 표1
INNER JOIN  주문내역 AS 표2
ON          표1.고객번호 = 표2.고객번호
```
</br>

### INNER JOIN 규칙
1. 두 테이블 모두에 있는 데이터만 출력
2. 첫번째 테이블과 두번째 테이블 둘 다 있는 데이터라면 "중복"된 데이터가 있는 열도 다 그대로 출력

### 실전을 위한 SQL 무료 강의 (14/18) : Left Join
</br>

- LEFT JOIN은 테이블1에 있는 모든 행을 출력
- ON 조건이 맞지 않는 경우 오른쪽 테이블의 값은 NULL로 출력됨
```sql
SELECT     테이블1.열이름, 테이블2.열이름
FROM       테이블 1
LEFT JOIN  테이블 2
ON         테이블1.열이름 = 테이블2.열이름
```
</br>

### LEFT JOIN 규칙
1. 왼쪽 테이블에 있는 데이터만 출력 (LEFT JOIN 전에 쓰는 테이블, 즉 FROM 뒤에 쓰는 테이블)
2. 왼쪽 테이블에 있는 데이터라면 "중복"된 데이터가 있는 열도 다 그대로 출력
3. LEFT JOIN은 LEFT OUTER JOIN과 같은 말

</br>

### 실전을 위한 SQL 무료 강의 (15/18) : Right Join
</br>

- RIGHT JOIN은 테이블2에 있는 모든 행을 출력
- ON 조건이 맞지 않는 경우 왼쪽 테이블의 값은 NULL로 출력됨
- RIGHT JOIN은 실전에서 거의 사용하지 않는다고 봐도 무방함 (LEFT JOIN 반대이기 때문)
- RIGHT JOIN은 RIGHT OUTER JOIN과 같은 말
```sql
SELECT      테이블1.열이름, 테이블2.열이름
FROM        테이블 1
RIGHT JOIN  테이블 2
ON          테이블1.열이름 = 테이블2.열이름
```
</br>

### 실전을 위한 SQL 무료 강의 (16/18) : Full Join
</br>

- FULL JOIN은 테이블1, 2에 있는 모든 행을 출력
- FULL JOIN은 FULL OUTER JOIN과 같은 말
```sql
SELECT     테이블1.열이름, 테이블2.열이름
FROM       테이블 1
FULL JOIN  테이블 2
ON         테이블1.열이름 = 테이블2.열이름
```
