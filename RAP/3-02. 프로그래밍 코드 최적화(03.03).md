### 1) CDS Value Help usage 옵션 기능
- `usage` 는 Value Help에서 필터로 쓸지, 선택값을 되돌려줄지 동작 방식을 정하는 옵션
- 미지정 (= `#FILTER`) → WHERE <VH필드> = <현재값> 로 목록만 필터됨
-  `#RESULT` → <현재필드> = <선택값> 으로 선택 후 자동세팅됨
-  `#FILTER_AND_RESULT` → WHERE … + <현재필드> = <선택값> 둘 다 수행됨
```abap
@Consumption.valueHelpDefinition: [{
    entity: {
        name: '<ValueHelp_Entity_Name>',      // 사용할 Value Help CDS
        element: '<ValueHelp_Key_Field>'      // 선택 시 <현재필드>에 들어갈 Key
    },
    additionalBinding: [{
        localElement: '<현재뷰_필드>',         // 현재 Projection View 필드
        element: '<ValueHelp_필드>',          // Value Help CDS 필드
        usage: #FILTER                       // WHERE <ValueHelp_필드> = <현재뷰_필드값>
        // usage: #RESULT                    // <현재뷰_필드> = <선택한 ValueHelp_필드값>
        // usage: #FILTER_AND_RESULT          // WHERE 조건 + 자동세팅 둘 다
        // 미지정 = 기본 #FILTER
    }]
}]
<FieldName>                                   // Value Help 적용 대상 필드
```
```abap
// Data Definitions - ZDSFLIGHT_CR_B03

define root view entity ZDSFLIGHT_CR_B03 as projection on ZDSFLIGHT_IR_B03
{

    @Search.defaultSearchElement: true

        entity: { name: '/DMO/I_Connection_StdVH', element: 'ConnectionID' },
        additionalBinding: [{ 
            localElement: 'Carrid', element: 'AirlineID', usage: #FILTER
        }]
    }]
    key Connid,
}
```

### 6) CDS `@UI.facet` – Object Page 섹션 구성 설정 
- Object Page의 섹션(블록) 구조를 정의하는 레이아웃 설정이며, `type` 값에 따라 표시 방식이 달라짐

| type 값 | 의미 | 화면에서의 역할 |
|---------|------|----------------|
| #IDENTIFICATION_REFERENCE | 기본 정보 묶음 영역 | Object Page 상단 대표 정보 표시 |
| #FIELDGROUP_REFERENCE | 필드 그룹 영역 | 관련 필드를 하나의 블록으로 표시 |
| #LINEITEM_REFERENCE | 테이블 영역 | 하위 데이터 목록을 표로 표시 |
| #CHART_REFERENCE | 차트 영역 | 데이터를 그래프로 시각화 |
| #DATAPOINT_REFERENCE | KPI 영역 | 핵심 숫자 값을 강조 표시 |
| #CONTACT_REFERENCE | 연락처 영역 | 담당자 및 연락처 정보 표시 |
| #COLLECTION | 그룹 컨테이너 | 여러 facet을 하나의 섹션으로 묶음 |

| purpose 값 | 의미 | 화면 위치 |
|------------|------|-----------|
| #STANDARD  | 일반 본문 섹션 | Object Page 본문 영역 |
| #HEADER    | 헤더 전용 섹션 | Object Page 상단 영역 |


```abap
annotate view <View_Name> with
{
  @UI.facet: [
    {
      id: '<Facet_ID>',                     // 섹션 고유 ID (필수)
      type: #IDENTIFICATION_REFERENCE,      // 섹션 타입 (필수)

      purpose: #STANDARD,                   // #STANDARD / #HEADER
      label: '<Section_Title>',             // 화면에 표시될 제목
      position: <Facet_Position>,           // 표시 순서

      parentId: '<Parent_Facet_ID>',        // #COLLECTION 안에 포함될 때 사용
      targetQualifier: '<Qualifier>',       // identification/fieldGroup 연결
      targetElement: '<Association>',       // LINEITEM_REFERENCE 등에서 사용
      hidden: false                         // true면 섹션 숨김
    }
  ]

  @UI.identification: [
    {
      qualifier: '<Qualifier>',             // facet의 targetQualifier와 동일
      position: <Field_Position>
    }
  ]
  <FieldName>;
}
```

### 7) LineItem 색상(criticality) 지정
- 색상은 `criticality` 에 연결된 숫자 필드(1/2/3) 값으로 결정됨
- 색상 계산용 필드는 화면에 안 보이도록 `@UI.hidden: true` 처리
```abap
@UI.lineItem: [{
    position: <위치>,
    label: '<라벨>',
    criticality: '<Criticality필드명>',          // 색상 결정 필드 (1/2/3)
    criticalityRepresentation: #WITH_ICON      // #WITH_ICON / #WITHOUT_ICON
}]
<표시필드>;
```
```abap
// Data Definitions - ZDSFLIGHT_IR_B03

@AccessControl.authorizationCheck: #NOT_REQUIRED
@EndUserText.label: 'Interface view (root)'
@Metadata.ignorePropagatedAnnotations: true
define root view entity ZDSFLIGHT_IR_B03
  as select from sflight
  composition [0..*] of ZSBOOK_I_B03 as _Item
  association to scarr               as _Scarr on $projection.Carrid = _Scarr.carrid
{
      case
        when $projection.Seatsremain = 100 then 3 // green
        when $projection.Seatsremain > 20 then 2 // yellow
        when $projection.Seatsremain > 0 then 1 // red
        else 0 // unknown
      end                 as SeatsremainCriticality, // 색상코드 필드

      cast(
             cast( seatsocc as abap.fltp ) / cast( seatsmax as abap.fltp ) * 100.0
             as abap.dec( 15, 2 )
         )                as Percentage,
      case
        when $projection.Percentage > 90 then 3 // green
        when $projection.Percentage > 50 then 2 // yellow
        when $projection.Percentage > 0 then 1 // red
        else 0 // unknown
      end                 as PercentageCriticality, // 색상코드 필드
}
```

### 8) `@UI.dataPoint` 설정
- 단순 컬럼이 아니라 의미 있는 수치(KPI, 상태값, 진행률 등)를 강조해서 보여줄 때 사용
- `qualifier` 로 facet과 연결하고, `criticality` 로 색상이나 Progress 형태까지 제어 가능
- `type: #AS_DATAPOINT` 는 List Report 컬럼을 단순 숫자가 아니라 KPI/Progress/색상 적용된 DataPoint 형태로 표시하고 싶을 때 사용
```abap
@UI.dataPoint: {
    title: '<표시 제목>',                   // KPI 제목
    qualifier: '<Qualifier명>',             // facet과 연결용 qualifier
    visualization: #PROGRESS,               // #PROGRESS (진행률 바)
                                            // #RATING   (별점 형태)
                                            // #DONUT    (도넛 차트)
                                            // (미지정 시 기본 숫자 강조 표시)
    targetValue: 100,                       // 목표 기준값 (Progress에서 주로 사용)
    criticality: '<Criticality필드명>'       // 1=Red, 2=Yellow, 3=Green
}
<표시필드>;

@UI.hidden: true                            // 색상 계산용 필드 숨김
<Criticality필드명>;
```
